## Ze Notes とは

Ze Notes は、Zotero のノートと注釈をテーブルで管理・可視化するためのプラグインです。添付ファイルのノートや注釈を使って、Zotero 上で直接システマティックレビューを行いたい場合に役立ちます。コレクションごとに1つのテーブルが作られ、行はアイテム、列はタグ、セルにはそのタグが付いたノートと注釈が入ります。ノートはテーブル上で直接編集でき、新しいノートを追加することもできます。

ノートと注釈は Zotero の中に保存されたままです。Ze Notes はそれらを読み取って表示し、編集内容を Zotero を通じて書き戻します。研究データの複製を別に管理する必要はありません。

ノート管理のほかに、次の機能があります。

- Google または DeepL によるノート・注釈の翻訳
- セル、行、列、テーブル全体、またはセル内の選択部分に対する生成AIプロンプトの実行
- Tesseract OCR による注釈・ノート内の画像のテキスト化(結果のAI補正も可能)
- テーブルの DOCX、XLSX、XLS、HTML、Markdown へのエクスポート

翻訳とAIは Zotero の PDF リーダー内からも利用できます。

## 前提条件

**必須:**

- Zotero 7、8、または 9(最小バージョン 7.0.0-beta.70)

**使いたい機能に応じて任意:**

- OCR 用の [Tesseract](https://tesseract-ocr.github.io/tessdoc/Downloads.html) のインストール
- 使用する翻訳・AIサービスの API キー(DeepL、OpenAI、Gemini、DeepSeek、Claude、またはカスタムプロバイダ)

## インストール

1. リリースページにアクセスします: https://github.com/frianasoa/Ze-Notes/releases/
2. 最新の `.xpi` ファイルをダウンロードします。ブラウザがインストールを試みて失敗する場合は、リンクを右クリックして「名前を付けて保存」を選んでください。
3. Zotero で「ツール → アドオン」(Zotero のバージョンによっては「プラグイン」)を開きます。
4. `.xpi` ファイルをそのウィンドウにドラッグします。または、右上の歯車アイコンをクリックして「Install Add-on From File」を選び、ファイルを指定します。
5. Zotero を再起動します。

Zotero プラグイン全般については https://www.zotero.org/support/plugins を参照してください。

## はじめに

1. Zotero で、ファイルをコレクション(フォルダ)に入れて管理します。
2. アイテムにノートを追加するか、添付ファイルに注釈や注釈コメントを付けます。ノートまたは注釈にタグを付けます。タグがテーブルの列になります。
3. コレクションを右クリックして「Show notes」を選びます。同じ操作は「ツール → Ze Notes → Show notes」からも実行でき、グループやライブラリ全体を選択した状態でも機能します。
4. ノートと注釈を含むテーブルのタブが開きます。

![image](./docs/images/screenshot-main-01.png)

主な操作は2つのコンテキストメニューから行います。

- **セルのコンテキストメニュー。** セルを右クリックすると、その内容に対する操作が表示されます: エントリの表示、ノートの編集、注釈コメントの編集、ノートの作成・削除、翻訳、AIプロンプトの実行、画像のOCR、添付ファイルの表示。
- **見出しのコンテキストメニュー。** 列見出しを右クリックすると、テーブル全体に対する操作が表示されます: 並べ替え、列・行の表示/非表示、エクスポート、Dropbox、設定。

設定は「編集 → 設定」または「ツール → Ze Notes → Show settings」、および見出しのコンテキストメニューから開けます。

## テーブル

**編集。** ノートと注釈コメントはテーブル上で直接編集できます。セルのコンテキストメニューから新しいノートを作成したり、不要になったノートを削除したりすることもできます。

**レイアウト。** 列をドラッグして並べ替え、列幅を変更し、列や行を非表示/再表示できます。非表示にした行と列はコレクションごとに記憶されます。見出しのコンテキストメニューの「Reset default widths」で元の列幅に戻せます。タグのないノートと注釈は「Untagged」列に入り、このラベルは設定で変更できます。

**フィルタ。** 見出しのコンテキストメニューの「Filter data」で、すべて表示、タグ付きのみ表示、タグなしのみ表示を切り替えられます。

**並べ替え。** 列ごとに昇順・降順で並べ替えられ、テーブルソートダイアログで複数の列を組み合わせられます。テーブル表示設定の「legacy display」オプションで旧バージョンのレイアウトに戻せます。また、列ソーターを常に表示しておくこともできます。

**外観。** 列ごとに背景色と文字色を設定でき、列見出しに接頭辞や接尾辞を追加できます。

**検索。** 検索バーでテーブル内のテキストを検索できます。

**キーボードショートカット:**

| ショートカット | 操作 |
|----------|--------|
| Ctrl+F | 検索バーを開く |
| Enter / F3 | 次の一致へ |
| Shift+Enter / Shift+F3 | 前の一致へ |
| Escape | 検索バーを閉じる |
| Ctrl+Plus | ズームイン |
| Ctrl+Minus | ズームアウト |

## 翻訳

Ze Notes は、ノート、注釈、注釈コメントを次の2つのサービスで翻訳できます。

- **Google翻訳。** キーなしで動作します。設定で Cloud Translation API キーを指定するとそれを使用し、指定がなければ無料エンドポイントにフォールバックします。
- **DeepL。** DeepL API キーが必要です。

設定で翻訳先の言語と必要なキーを指定してください。設定が済むと、翻訳の操作がテーブルのコンテキストメニューと PDF リーダーに表示されます。

## 生成AI

ノートと注釈に対してAIプロンプトを実行できます。プロンプトの対象は、セル1つ、行、列、テーブル全体、またはセル内の選択部分です。各プロバイダの操作は、API キーを設定するとコンテキストメニューに表示されます。

コンテキストメニューのAI操作の隣にある「Ai data settings」ダイアログで、AIに送信するデータの範囲(引用、出典、著者、および個々のタグ・注釈パート)を選べます。選択内容はコレクションごとに記憶されます。

**組み込みプロバイダ:**

| プロバイダ | 設定項目 |
|----------|---------------|
| OpenAI | APIキー、モデル、最大トークン |
| Gemini | APIキー、モデル |
| DeepSeek | APIキー、モデル |
| Claude (Anthropic) | APIキー、モデル、最大トークン |
| Custom AI | URL、APIキー、モデル、リクエストオプション、レスポンス整形関数 |

各プロバイダにはシステムメッセージとユーザープロンプトを設定でき、満足のいく結果になるまで調整できます。

**Custom AI。** 上記以外のサービスを使う場合や、同じサービスに対して複数の定型プロンプトを用意したい場合は、Custom AI 設定でリクエスト自体を記述できます: エンドポイントの URL、`${variable}` プレースホルダを使った JSON のリクエストオプション、レスポンスから結果を取り出す小さな関数です。複数の構成を保存して、どれをコンテキストメニューに表示するか選べます(同時に表示されるのは最大20件)。実際に動作する例は[ドキュメント](https://frianasoa.github.io/Ze-Notes/)を参照してください。

**キーとコストについて。** APIキーはお使いのマシン上の暗号化されたデータベースにローカル保存されます。利用料金は Ze Notes ではなくプロバイダから請求されるため、プロバイダのダッシュボードを確認し、そちらで支出上限を設定してください。

## TesseractによるOCR

Tesseract がマシンにインストールされていれば、Ze Notes は注釈やノート内の画像をテキストに変換できます。

1. Tesseract をインストールします: https://tesseract-ocr.github.io/tessdoc/Downloads.html
2. Ze Notes は Windows、Linux、macOS の標準的な場所から Tesseract の実行ファイルを探します。別の場所にインストールした場合は、設定でパスを指定してください。
3. 利用可能な言語リストから認識言語を選びます。

設定が済むと、注釈画像とノート画像のコンテキストメニューにOCRの操作が表示されます。

**AI補正。** OCR の出力は完璧とは限りません。結果を設定済みの任意のAIプロバイダに通すことで、テキストの言語と構造を保ったまま認識エラーを修正できます。

## PDFリーダーとの連携

一部の操作は、Zotero の PDF リーダー内で、ハイライトしたテキストのコンテキストメニューから直接実行できます。

- ハイライトを Google または DeepL で翻訳する
- ハイライトに対して OpenAI または保存済みの Custom AI 構成でAIプロンプトを実行する

結果は、どのサービスが生成したかを示す `[[Title]]` マーカー付きで注釈コメントに追記されます。

## エクスポート

作業が終わったら、見出しまたはセルのコンテキストメニューから「Export」を選びます。2つのタブを持つエクスポートダイアログが開きます。

- **Select data。** 含める要素を選びます: 引用、出典、著者、および個々のタグ・注釈パート。「Check all」で一括切り替えできます。
- **Advanced settings。** テーブルスタイル、ハイライト、アイコン、メイン/サブの凡例の削除、空要素の除外、およびエクスポートファイルの隣にリンクされたリソース用のフォルダを作成するかどうかを設定します。

「Export」をクリックしてファイル名を指定します。選んだ拡張子が形式を決めます。

- Microsoft Word (`.docx`)
- Microsoft Excel (`.xlsx` と旧形式の `.xls`)
- HTML/XHTML (`.html`、`.xhtml`)
- Markdown (`.md`)

エクスポートが完了するとファイルが開きます。エクスポートのオプションはコレクションごとに記憶されます。

## Dropboxでの共有

Dropbox アプリの認証情報(クライアントIDとシークレット)を設定すると、Ze Notes がアカウント接続の手順を案内し、見出しのコンテキストメニューに2つの操作が表示されます。

- **Upload to dropbox。** 現在のコレクションをノート・添付ファイル・注釈ごとエクスポートして zip 化し、ダイアログに入力した共同研究者のメールアドレスを名前とする Dropbox フォルダにアップロードします。ダイアログには添付ファイル数、合計サイズ、推定処理時間が表示され、エクスポート速度のスライダーで調整できます。
- **Download from dropbox。** 自分の Zotero アカウントのユーザー名を名前とするフォルダに共有されたファイルの一覧を表示します。ファイルをダウンロードするとコレクションが Zotero にインポートされ(新規アイテムのみ追加)、インポートされた PDF 添付ファイルから注釈が更新されます。

また、Dropbox 設定を暗号化ファイルとしてエクスポートし、エクスポート時に生成されるパスワードとともに共同研究者へ送ることもできます。受け取った相手は、一から設定する代わりに自分の設定画面からそのファイルを読み込めます。

## データの保存場所

- **ノートと注釈**は従来どおり Zotero 内にあります。Ze Notes はそれらを移動も複製もしません。
- **プラグインの設定**は Zotero とは別の小さな SQLite データベースに保存されます。APIキーや、コレクションごとのテーブル設定(列の順序、非表示の行と列、色、並び順、エクスポートオプション)が含まれます。
- APIキーなどの**機密設定**はこのデータベース内で暗号化されます。
- **設定は同期されません。** Zotero データベースの外にあるため、同じ Zotero アカウントでログインしたコンピュータ間でも設定は同期されません。前述の暗号化エクスポート/インポートは、現在 Dropbox 設定について利用できます。

## ソースからのビルド

このプラグインは TypeScript と React で書かれ、esbuild でバンドルされています。

```bash
git clone https://github.com/frianasoa/Ze-Notes.git
cd Ze-Notes
npm install
npm run build
```

`npm run build` は最小化されたバンドルを生成し、`npm run dev` は開発用ビルドを生成します。出力先は `app/chrome/core/engine.js` です。
